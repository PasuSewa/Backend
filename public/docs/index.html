    Introduction

    This documentation aims to provide all the information you need to work with our API.

    As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

    Base URL

    https://pasunashi-backend.herokuapp.com

    Authenticating requests

    This API is not authenticated.

    Auth

    Send Code By Email

    This method will send the security code needed for the user to login by the "login with email code" method.

    Remember that the email parameter must exists either on the "emails" or in "recovery_emails" columns

    Example request:
    curl --request POST \
    "https://pasunashi-backend.herokuapp.com/api/send-code-by-email" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "Accept-Language: es | en | jp" \
    --data "{
    \"email\": \"kpejchzvyplqxgekzifwwbaltrnvgmvxtdbzpgnqlynwmldmizddedkutbimyfzojbelltpbynsdaynhecnmamfekrrgksqruqiyltlvdoyyihazjnfdsrfmazsieiunptjxghuobs\",
    \"isSecondary\": false
}"

    const url = new URL(
    "https://pasunashi-backend.herokuapp.com/api/send-code-by-email"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
    "Accept-Language": "es | en | jp",
};

let body = {
    "email": "kpejchzvyplqxgekzifwwbaltrnvgmvxtdbzpgnqlynwmldmizddedkutbimyfzojbelltpbynsdaynhecnmamfekrrgksqruqiyltlvdoyyihazjnfdsrfmazsieiunptjxghuobs",
    "isSecondary": false
}

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

    Example response (200):

    
{
    "status": 200,
    "message": "Succes!",
    "data": {}
}

    Example response (401, validation failed):

    Refresh 2-Factor-Authentication Secret

    requires authentication

    This method is responsible for updating and returning the secret key for the time-based one-time-password (the code generated in apps like Google Authenticator). In order to check if the user has set up correctly the new secret key, you can verify it using the endpoint of Register, Third Step.

    You can generate a QR-code for the user to scan using the structure given in the README file.
    Example request:
    curl --request GET \
    --get "https://pasunashi-backend.herokuapp.com/api/auth/refresh-2fa-secret" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "Accept-Language: es | en | jp"
    const url = new URL(
    "https://pasunashi-backend.herokuapp.com/api/auth/refresh-2fa-secret"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
    "Accept-Language": "es | en | jp",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

    Example response (200):

    
{
     "status": 200,
     "message": "Succes!",
     "data": {
         "secret": "AAAAAAAAAA",

    Grant Access to Confidential Information

    requires authentication

    This method may return a decrypted Credential, you can see the Credential interface on the README file.

    "https://pasunashi-backend.herokuapp.com/api/auth/grant-access" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "Accept-Language: es | en | jp" \
    --data "{
    \"accessTo\": \"impedit\",
    \"credentialId\": 9,
    \"accessingDevice\": \"similique\",
    \"accessingPlatform\": \"bmvk\",
    \"accessingPlattform\": \"eos\"
}"

    Request      

    POST api/auth/grant-access

    Body Parameters

    accessTo  string  

    Must be either one of these exact two options: "user-data" or "credential-data".

    credentialId  integer optional  

    Its required, only if "accessTo" is equal to "credential-data".

    accessingDevice  string  

    The user agent of the navigator, or the unique ID of the device. (min: 1, max: 190 char).

    accessingPlatform  string  

    Must be one of mobile, web, or desktop Must be at least 3 characters. Must not be greater than 7 characters.

    accessingPlattform  string  

    Must be one of these exact three options: "web", "desktop", "mobile".
